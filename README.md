把素材图片拖到 DDS 贴图上、调好位置和角度、一键导出覆盖原文件——整个流程不离开一个窗口，编辑记录随时可续。

## 功能概览

| 功能 | 说明 |
|------|------|
| **打开 Mod 文件夹** | 自动扫描 mod 内所有 DDS，缩略图列表里点选 |
| **模型库** | 设一个模型库根目录，按「模型 → 多个 mod（压缩包/文件夹）」浏览，带预览缩略图、一键打开切换 |
| **压缩包直接导入** | 选中 `.zip/.rar/.7z` 形式的 mod，内置 7-Zip 自动解压到暂存目录后按普通 mod 处理 |
| **拖拽 / 两步点击放置** | 把素材拖到画布；或点选素材 → 定中心点 → 拖鼠标定大小/角度 → 固定 |
| **镜像插入** | 勾选「镜像插入」，放置时自动水平翻转 |
| **图层调整** | 移动、缩放、旋转、调透明度、调混合模式、层序上下移、隐藏/删除 |
| **四角透视变形** | 选中图层 → 工具栏「四角变形」，拖动四个角点做自由透视 |
| **一键合并图层** | 多图层时可合并为一层，大幅提升操作流畅度 |
| **导出 DDS** | 合并画布并调用 texconv 编码，支持 BC1/BC3/BC5/BC7 多种格式，覆盖 mod 副本 |
| **恢复上一次 / 完全恢复** | 每次导出自动备份前一版；也可一键恢复到最初的原始贴图 |
| **编辑记录自动保存** | 未合并的图层工程保存在工作目录，下次打开同一 mod 自动续编 |
| **素材库** | 按使用频率排序，启动自动加载上次记住的文件夹 |
| **缩放与平移** | 滚轮缩放、中键/空格拖拽平移、底栏快捷按钮 |
| **压缩 RAR（时间戳可定制）** | 把输出目录里的 mod 副本一键打包成 RAR；时间戳位置（前/后）与格式可在设置里自定义 |
| **窗口置顶** | 方便边看参考图边编辑 |

## 快速上手

### 1. 下载并运行

从 [Releases](../../releases) 页面下载，提供两种版本，任选其一：

| 版本 | 文件 | 适合 | 说明 |
|------|------|------|------|
| **安装版**（推荐） | `ZZZ-DDS-Editor-*-setup.exe` | 长期使用 | 安装到本地，创建桌面 / 开始菜单快捷方式；**启动更快**（不需每次自解压） |
| **便携版** | `ZZZ-DDS-Editor-*-portable.exe` | 免安装 / 拷着走 | 单文件双击即用，不写注册表；**每次启动会先自解压到临时目录，冷启动略慢** |

> 两个版本功能完全相同。图省事、随身携带用便携版；日常常用、想启动更快就用安装版。
> 首次打开若被 Windows SmartScreen 拦截，点「更多信息 → 仍要运行」即可（因为安装包未做数字签名）。

### 2. 第一次使用：设置目录

点右上角 **⚙ 设置**，配置两个目录：

- **工作目录**：编辑记录（分层工程文件 + 原图副本）保存到这里，建议新建专用文件夹，例如 `D:\zzz贴图记录`
- **输出目录**：打开 mod 后会在这里生成 mod 副本；导出 DDS 只覆盖这个副本，**原始 mod 文件夹永远不会被改动**

> 未设工作目录时，⚙ 按钮会有蓝色描边提示。

### 3. 打开要编辑的贴图

1. 点 **打开 Mod 文件夹**，选择 mod 的根目录
2. 弹出缩略图选择框，点一张 DDS 开始编辑
3. 下次想继续，直接点 **最近记录**，或再次打开同一 mod 文件夹 → 自动接上上次的图层

### 4. 添加素材

- 点右侧「待插入贴图」面板的 **选择文件夹**，选放素材（PNG / DDS）的文件夹
- 素材按使用频率自动排序；可在设置里记住文件夹，下次启动自动加载

### 5. 放置素材（两种方式）

**拖拽**：直接把右侧素材拖到画布，松手处即为图层中心。

**两步点击**（适合精确定位 / 定角度）：
1. 在右侧**单击**一个素材（高亮选中）
2. 在画布上**点第一下** → 定中心点
3. **移动鼠标** → 实时预览大小和旋转角度（当前鼠标位置 = 右下角）
4. **点第二下** → 固定；`Esc` 随时取消

放置后仍可选中继续调整位置 / 缩放 / 旋转 / 四角变形。

### 6. 导出

1. 在工具栏选择导出格式（推荐「**跟随原图格式**」保持原大小；彩色高质量选 **BC7 sRGB**）
2. 点 **导出 DDS（覆盖输出副本）**
3. 状态栏会显示导出前后的格式和文件大小

> 导出只写输出目录里的 mod 副本，原始 mod 文件夹不受影响。  
> 导出前自动保存前一版备份，可随时点「**恢复上一次**」撤销；「**完全恢复**」还原到最初的原始贴图（有确认弹窗）。

### 7. 打包 Mod

编辑完成后，点工具栏 **压缩 RAR**，把输出目录里的整个 mod 副本打包为带时间戳的 RAR 文件，直接发给别人或上传到 mod 站使用。

> 时间戳可在 **⚙ 设置 → 压缩包时间戳** 里自定义：加在 mod 名【前面】还是【后面】，格式支持 `YYYY YY MM DD HH mm ss` 占位符（例如 `YYYY-MM-DD_HH-mm-ss`），设置面板里有实时示例预览。

## 模型库（管理同一模型的多个 mod）

同一个角色/模型往往有很多套 mod，模型库让你集中浏览、预览、一键切换编辑：

1. **⚙ 设置 → 模型库目录** 选一个根目录（也可在「模型库」面板里直接选）。
2. 点工具栏 **模型库** 打开面板：左侧是模型列表，右侧是该模型下的各个 mod 卡片（带预览图）。
3. 点卡片 **打开** → 若是压缩包会自动解压到暂存目录，再进入正常的选贴图 / 编辑 / 导出流程。

**目录结构约定**（两级）：

```
<模型库根目录>/
├── <模型A>/               ← 一个模型（角色）
│   ├── modX.zip          ← 一个 mod（zip / rar / 7z 压缩包）
│   ├── modY/             ← 或一个 mod 文件夹
│   └── modX.png          ← 可选：与 mod 同名的预览图（手动兜底）
└── <模型B>/ ...
```

**预览图来源**：优先用压缩包/文件夹内的预览图（`preview.*` 或第一张图片）；若没有，可点卡片上的 **指定预览图** 手动选一张外部图片（会拷成同名 sidecar，下次自动认出）。散放在根目录下的压缩包会归入「未分组」。

## 记录文件结构

```
<工作目录>/
├── <mod文件夹名>/
│   ├── <贴图名>.dds        ← 原图副本（导出永不覆盖）
│   ├── <贴图名>.zzzproj    ← 分层工程，可续编
│   └── <贴图名>.prev.dds   ← 上一次导出前的版本（「恢复上一次」用）
└── _extracted/            ← 模型库里压缩包 mod 的解压暂存

<输出目录>/
└── <mod文件夹名>/          ← mod 完整副本，导出 DDS 只改这里
    └── ...
```

## 导出格式说明

| 格式 | 适用场景 | 说明 |
|------|----------|------|
| 跟随原图格式 | **推荐默认** | 保持原 DDS 的压缩格式，文件大小不变 |
| BC7 sRGB | 彩色贴图，追求最高质量 | 体积约为 BC1 的 2 倍 |
| BC7 线性 | 法线图、数据纹理 | 线性空间，无 gamma 编码 |
| BC3 sRGB | 带 alpha 的彩色贴图 | DXT5，兼容性好 |
| BC1 sRGB | 无 alpha 的彩色贴图 | 最小体积，无透明度 |
| BC5 线性 | 法线图（双通道） | RG 压缩，精度高 |

## 技术栈

- **Electron** — 桌面应用框架
- **electron-vite** — 构建工具
- **TypeScript** — 全栈类型安全
- **Fabric.js v6** — 画布图层编辑引擎
- **dds-ktx-parser** — DDS 文件解析（BC1/3/5/7 解码）
- **texconv.exe**（Microsoft DirectXTex）— DDS 编码，内嵌于 `resources/texconv/`
- **7-Zip**（7z.exe + 7z.dll，含 RAR 编解码）— 解压模型库里的 mod 压缩包，内嵌于 `resources/7zip/`

## 开发者：从源码构建

### 环境要求

- **Node.js v18+**（推荐 v22）
- **Windows x64**（texconv.exe 仅 Windows 可用；macOS 下可开发，导出会 fallback 为未压缩 DDS）

### 构建步骤

```bash
# 安装依赖
npm install

# 开发模式（热重载）
npm run dev

# 一键构建 portable exe
bash build.sh
```

构建完成后，会同时产出两种安装包（文件名都带 `package.json` 里的版本号）：
- 根目录：`ZZZ-DDS-Editor-安装版-v<版本>.exe` 和 `ZZZ-DDS-Editor-便携版-v<版本>.exe`
- `release/`：`ZZZ-DDS-Editor-<版本>-setup.exe` 和 `ZZZ-DDS-Editor-<版本>-portable.exe`

> **升级版本号**：先改 `package.json` 的 `"version"`，再跑 `bash build.sh`——文件名会自动带上新版本号，根目录里旧版本号的 exe 会被清掉（正在运行而占用的会跳过）。

> **注意**：构建前请先关闭正在运行的编辑器，否则根目录同名 exe 被占用，`build.sh` 最后的复制步骤会失败（新构建仍保留在 `release/` 目录）。

### 项目结构

```
src/
├── main/           # Electron 主进程
│   ├── ipc.ts      # 所有 IPC 处理器
│   ├── records.ts  # 工作目录记录管理、备份/恢复
│   ├── modLibrary.ts # 模型库扫描 / 7z 解压 / 预览提取 / 暂存
│   ├── ddsWrite.ts # 无压缩 sRGB DDS 构建（texconv 输入）
│   ├── ddsHeader.ts / ddsThumb.ts  # DDS 元信息 + 缩略图解码
│   ├── texconv.ts  # texconv.exe 调用封装
│   └── settings.ts # 持久化设置
├── renderer/
│   └── src/
│       ├── main.ts       # 渲染进程主逻辑（状态管理、IPC 调用编排）
│       ├── editor.ts     # Fabric.js 画布封装（图层、放置、透视、导出）
│       ├── ddsDecode.ts  # DDS → Canvas 解码（opaque base + alpha 分离）
│       ├── library.ts    # 素材面板
│       ├── modLibraryPanel.ts # 模型库面板（模型/mod 浏览 + 预览）
│       ├── layersPanel.ts / modPicker.ts / settingsPanel.ts
│       └── progress.ts   # 状态栏进度条
├── preload/        # contextBridge API 暴露
└── shared/
    └── types.ts    # 主进程/渲染进程共享类型定义
resources/
├── texconv/
│   └── texconv.exe # Microsoft DirectXTex 编码工具
└── 7zip/
    └── 7z.exe + 7z.dll # 解压 zip/rar/7z 的 mod 压缩包
```

## 已知限制

- 仅支持 Windows x64（texconv.exe 依赖）
- DDS 解码依赖 dds-ktx-parser，支持 BC1/BC3/BC5/BC7；其他格式（如 BC4、ASTC）缩略图可能无法显示
- 压缩 RAR 功能依赖系统已安装 WinRAR（解压 mod 压缩包则用内置 7-Zip，无需额外安装）
